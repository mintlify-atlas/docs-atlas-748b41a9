The DEC backend is a REST API built with **Node.js** and **Express** that powers the entire DEC (Detector de Enfermedades en Cafetales) platform. It runs on **port 8089** by default and provides JWT-based authentication, persistent data storage via **MongoDB**, AI-driven disease detection history, and **Cloudinary** image storage for plant photos. All client requests — from the React Native mobile app and the web admin dashboard — are served through this API.

## Prerequisites

Before installing, ensure you have the following available:

- **Node.js v18+** and npm — the runtime for the Express server
- **MongoDB** — a local instance (`mongod`) or a hosted [MongoDB Atlas](https://www.mongodb.com/atlas) cluster
- **Cloudinary account** — used to store and serve plant detection images (JPG/PNG, up to 10 MB)
- **Gmail credentials** — a Gmail address and App Password used by Nodemailer to deliver contact-form messages; Resend handles verification and password-reset emails separately
- **Google Cloud Console credentials** — a Google OAuth 2.0 Client ID for Google Sign-In
- **Meta Developer credentials** — a Facebook App ID for Facebook Login

## Installation Steps

<Steps>
  <Step title="Navigate to the backend directory">
    From the repository root, move into the `backend/` folder:

    ```bash
    cd backend
    ```
  </Step>

  <Step title="Install Node.js dependencies">
    Install all required packages listed in `package.json`:

    ```bash
    npm install
    ```

    This installs Express, Mongoose, jsonwebtoken, bcryptjs, Cloudinary, google-auth-library, Nodemailer, Multer, and all other dependencies.
  </Step>

  <Step title="Create the .env file">
    Create a `.env` file in the `backend/` directory and populate it with your credentials. The server reads these values at startup via `dotenv`:

    ```bash
    # backend/.env

    # MongoDB connection string
    dbURI=mongodb://localhost:27017/dec

    # JWT signing secret — use a long, random string in production
    JWT_SECRET=your_super_secret_jwt_key_here

    # Cloudinary — image storage for plant detections and profile pictures
    CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
    CLOUDINARY_API_KEY=your_cloudinary_api_key
    CLOUDINARY_API_SECRET=your_cloudinary_api_secret

    # Google OAuth 2.0 — Client IDs for web and mobile
    GOOGLE_WEB_CLIENT_ID=your_google_web_client_id
    GOOGLE_APP_CLIENT_ID=your_google_app_client_id

    # Facebook Login
    FACEBOOK_APP_SECRET=your_facebook_app_secret

    # Resend — transactional email service for verification and recovery emails
    RESEND_API_KEY=your_resend_api_key
    FROM_EMAIL=no-reply@your-verified-domain.com

    # Gmail / Nodemailer — used for the contact-us form
    EMAIL_USER=your_email@gmail.com
    EMAIL_PASS=your_app_specific_password

    # Server port (optional — defaults to 8089)
    PORT=8089
    ```
  </Step>

  <Step title="Start the server">
    **Development mode** (auto-restarts on file changes via nodemon):

    ```bash
    npm run dev
    ```

    **Production mode**:

    ```bash
    npm start
    ```

    On a successful start you will see:

    ```
    conectado a la base de datos
    🗄️ DB activa: dec
    ✅ Servidor corriendo en puerto 8089
    ```
  </Step>
</Steps>

## Verify the Server

Once running, confirm the API is healthy by sending a `GET` request to the root route:

```bash
curl http://localhost:8089/
```

Expected response:

```
 Servidor funcionando correctamente
```

You can also open `http://localhost:8089/` in a browser — the same message will be displayed.

## Available Scripts

The following npm scripts are defined in `package.json`:

| Script | Command | Description |
|--------|---------|-------------|
| `start` | `node server.js` | Run the server with Node.js directly. Suitable for production deployments. |
| `dev` | `nodemon server.js` | Run the server with nodemon, which watches for file changes and restarts automatically. Recommended for local development. |

<Note>
  The server defaults to **port 8089** as defined in `server.js` (`process.env.PORT || 8089`). To run on a different port, set the `PORT` variable in your `.env` file or as an environment variable before starting the process:

  ```bash
  PORT=3000 npm start
  ```
</Note>
