DEC (Detección de Enfermedades en Cafetales) is an AI-powered, cross-platform application built to help **farmers and agronomists** detect diseases in coffee plants quickly and accurately. By pointing a smartphone camera at a coffee leaf, users receive an instant on-device diagnosis powered by a YOLOv11 model compiled to TensorFlow Lite — no internet connection required for inference. Detection results are stored locally, synced to the cloud, and surfaced in a web-based admin dashboard for operational oversight and reporting.

## What DEC Does

<CardGroup cols={2}>
  <Card title="On-Device AI Detection" icon="brain" href="/architecture">
    A YOLOv11 model exported as a TensorFlow Lite float16 binary runs entirely on the device using `react-native-fast-tflite`. Diagnoses are instant and work offline in the field, returning the detected pathology, confidence score, and recommended treatment.
  </Card>
  <Card title="Multi-Platform Support" icon="mobile-screen" href="/quickstart">
    A single React Native + Expo codebase targets iOS, Android, and modern web browsers. Platform-specific features (native camera, SQLite, secure token storage) are abstracted behind Expo's universal API layer.
  </Card>
  <Card title="Detection History & Sync" icon="clock-rotate-left" href="/architecture">
    Every detection is persisted to an on-device SQLite database first, then automatically synced to MongoDB through the REST API. Users can browse their full diagnosis history, view image thumbnails, and re-read treatment advice — even after reinstalling the app.
  </Card>
  <Card title="Admin Dashboard" icon="chart-line" href="/architecture">
    A web-only React Native Web interface provides administrators with KPI counters, incidence trend charts, a geographic heat map of detections, user management, and multi-format data exports — all secured behind JWT-authenticated admin routes.
  </Card>
</CardGroup>

## Supported Platforms

DEC ships as a **single unified codebase** that runs natively on **iOS** and **Android** via Expo's managed workflow, and as a **responsive progressive web app** in any modern browser. Production builds for iOS and Android are generated with Expo Application Services (`eas build`). The web build is produced via `expo export --platform web`. On mobile, the app uses the device camera and stores data in SQLite; on web, it falls back to `sessionStorage` for token management and the browser's `getUserMedia` API for camera access.
