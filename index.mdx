DEC (Detección de Enfermedades en Cafetales) is a cross-platform application that helps farmers and agronomists identify coffee plant diseases instantly. Point the camera at a leaf, and DEC's on-device TensorFlow Lite model — trained on YOLOv11 — returns a diagnosis with confidence score, severity rating, and treatment recommendations in seconds.

## How DEC Works

<Steps>
  <Step title="Authenticate">
    Users register or sign in with email/password, Google, or Facebook. The backend issues a JWT that secures every subsequent request.
  </Step>
  <Step title="Capture an image">
    The mobile app opens the device camera (or the image picker). The image is passed directly to the on-device TensorFlow Lite model — no upload required for inference.
  </Step>
  <Step title="AI analysis">
    The YOLOv11-based model classifies the disease, returns a confidence score, and maps the result to a pathology entry in the database for treatment details.
  </Step>
  <Step title="Save & sync">
    The detection is stored locally in SQLite and automatically synced to MongoDB via the REST API when a network connection is available.
  </Step>
</Steps>

## Key Features

<CardGroup cols={3}>
  <Card title="On-Device AI" icon="brain">
    TensorFlow Lite inference runs entirely on the device — no internet required for detection.
  </Card>
  <Card title="Cross-Platform" icon="mobile">
    Single React Native codebase targets iOS, Android, and responsive web via Expo.
  </Card>
  <Card title="Multi-Auth" icon="shield-check">
    Email/password, Google OAuth, and Facebook SDK with secure JWT session management.
  </Card>
  <Card title="Offline-First" icon="database">
    SQLite stores detections locally; background sync pushes them to MongoDB automatically.
  </Card>
  <Card title="Admin Dashboard" icon="chart-bar">
    Web-only panel with user management, detection approval, incidence heatmaps, and KPI tiles.
  </Card>
  <Card title="Dataset Export" icon="file-zipper">
    Admins can export approved detections as a ZIP archive for model retraining workflows.
  </Card>
</CardGroup>
